## Regeln

### R1 — Keine AI-Placeholder-Kommentare [REVIEWED] [MEDIUM]

**Kategorie:** cat-26-rule-1 | **Gewicht:** ×2 | **FixType:** code-fix

**Muster (spezifisch für AI-Template-Sprache):**
- `// TODO: implement`
- `// Add your logic here`
- `// This is a placeholder`
- `// Replace with actual`
- `// [placeholder]`
- `// Fill in the implementation`

**Abgrenzung zu cat-18-rule-6 (checkTodoComments):**
cat-18-rule-6 zählt ALLE TODOs/FIXMEs ohne Ticket-Referenz. Diese Regel zielt auf
spezifische AI-Template-Sprache — Kommentare, die unfertige KI-Generierung anzeigen,
nicht normale Entwickler-TODOs.

**Threshold:** >3 solcher Kommentare im Repo.

**Suggestion-Template:**
```
Cursor-Prompt: 'Find all placeholder comments (TODO: implement, Add your logic here, etc.)
in [file] and either implement them or create GitHub issues with #number references.'
```

---

### R2 — Keine AI-Tool-Fingerprints [ADVISORY] [INFO]

**Kategorie:** cat-26-rule-2 | **Gewicht:** ×1 | **FixType:** code-fix

**Muster:**
- `// Generated by Lovable`
- `// Copilot suggestion`
- `.bolt/prompt` Datei im Repo
- `/* AI-generated */`
- `// This file was generated`

**Hinweis:** Score 4 (nicht 0) — Fingerprint ≠ Problem, aber erfordert Review-Bestätigung.
Info-Finding, nicht blockierend.

---

### R3 — Keine Überkommentierung [ADVISORY] [INFO]

**Kategorie:** cat-26-rule-3 | **Gewicht:** ×1 | **FixType:** code-fix

**Check:** Verhältnis Kommentarzeilen zu Code-Zeilen > 40% in .ts/.tsx Dateien > 50 Zeilen.

**Warum:** KI kommentiert jeden trivialen Schritt ("// increment counter by 1").
Erfahrene Entwickler kommentieren nur nicht-offensichtliche Logik.

**Abgrenzung zu cat-2-rule-8 (checkCommentedOutCode):**
cat-2-rule-8 findet AUSKOMMENTIERTEN Code (deaktivierter Code-Block).
Diese Regel misst ERKLÄRUNGSKOMMENTARE-Quote (aktive Kommentare die Code beschreiben).

---

### R4 — Keine Placeholder-Credentials [REVIEWED] [HIGH/MEDIUM]

**Kategorie:** cat-26-rule-4 | **Gewicht:** ×3 | **FixType:** code-fix

**Muster:**
- `"YOUR_API_KEY_HERE"`
- `"INSERT_YOUR_TOKEN"`
- `"REPLACE_WITH_ACTUAL"`
- `"your-project-id"`
- `"YOUR_SECRET_HERE"`

**Ausnahmen (nicht geflaggt):** `.env.example`, `.env.sample`, `.md`, Test-Dateien, `/docs/`

**Severity:**
- `high` in `/src/` (direktes Produktionsproblem)
- `medium` in Konfigurationsdateien

**Abgrenzung zu cat-3-rule-30 (checkHardcodedSecrets):**
checkHardcodedSecrets findet ECHTE eingebettete Secret-Werte (z.B. `AIzaSy...`).
Diese Regel findet TEMPLATE-PLATZHALTERTEXT — unfüllte KI-Vorlagen.
Komplementär, nicht überlappend.

---

### R5 — Konsistente Kommentar-Sprache [ADVISORY] [INFO]

**Kategorie:** cat-26-rule-5 | **Gewicht:** ×1 | **FixType:** code-fix

**Check:** >20% Kommentare in Minderheitssprache bei >5 detektierten Kommentaren.

**Warum:** KI wechselt Kommentar-Sprache je nach Prompt-Sprache.
EN-Prompts → EN-Kommentare. DE-Prompts → DE-Kommentare.
Mischung deutet auf mehrere KI-Sessions mit verschiedenen Prompts.
